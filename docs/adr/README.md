# Architecture Decision Records

ADRs capture **why** we picked one approach over another, so future contributors
(including future-you) can tell which decisions were deliberate and which were
accidents of history.

Each ADR has a status:

- **Proposed** — under discussion
- **Accepted** — in force; the codebase reflects this
- **Superseded by ADR-NNNN** — replaced; kept for context

Write a new ADR when you make a decision that's hard to reverse, that surprised
you, or that future readers might be tempted to undo without context.
